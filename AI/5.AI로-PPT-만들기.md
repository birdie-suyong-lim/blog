## 들어가며 — "슬라이드 만드는 시간"이 가장 아까운 시대

요즘 슬라이드를 만들 때 가장 거슬리는 건 발표 *내용*이 아니라 **레이아웃·도형 정렬·아이콘 찾기·아이콘 색깔 맞추기** 같은 잡일이다. 핵심 메시지를 정리하는 데 30분, 그걸 슬라이드 형태로 옮기는 데 3시간. 비율이 거꾸로다.

AI 기반 PPT 도구는 이 비율을 뒤집어준다. 텍스트 한 단락만 던지면 **구조·디자인·시각 요소까지 한 번에 초안**을 만들어주고, 사람은 마지막 *판단*만 남긴다. 다만 도구마다 강점이 다르고, 무턱대고 쓰면 "AI가 만든 티 나는 슬라이드"가 그대로 발표 자료가 되기 십상이다.

이 글은 AI로 PPT를 만드는 두 가지 접근, 주요 도구의 위치, 그리고 **상황별로 어떤 도구를 어떤 순서로 쓰는 게 좋은지**를 정리한다.

---

## 1. 개념 — AI 슬라이드 생성에는 두 갈래가 있다

### 갈래 A: AI 네이티브 슬라이드 도구

처음부터 "텍스트 → 슬라이드"를 위해 만들어진 도구. Gamma, Tome, Beautiful.ai, Decktopus 등이 여기 속한다.
- 프롬프트 한 줄 또는 한 단락만 입력하면 **전체 덱**을 통째로 생성
- 디자인 템플릿이 AI 출력 결과와 묶여 있어 일관성이 좋음
- 단점: PowerPoint·Keynote의 정밀 편집 기능에는 못 미침. 회사 표준 템플릿 강제 적용이 어려움

### 갈래 B: 기존 PPT 도구 + AI 어시스턴트

PowerPoint·Google Slides·Canva 같은 **기존 도구 내부에** AI가 들어간 형태.
- Microsoft Copilot in PowerPoint, Google Slides + Gemini, Canva Magic Design
- 회사·학교에서 이미 PowerPoint를 표준으로 쓰는 환경에 자연스럽게 융합
- 단점: AI가 만든 슬라이드가 도구의 기존 템플릿 안에서 생성되어 **신선도는 떨어지지만 호환성·편집성은 우수**

### 갈래 C (보너스): LLM + 변환 도구

ChatGPT·Claude로 **텍스트 구조와 슬라이드 내용**만 만든 뒤, Marp·SlideSpeak·python-pptx 같은 변환기로 PPTX로 떨어뜨리는 방법.
- 가장 유연하지만 손이 많이 감
- 개발자 친화적, 자동화·반복 작업에 강함

## 2. 언제 어떤 갈래를 쓰는가

| 상황 | 추천 갈래 | 이유 |
|------|----------|------|
| 30분 안에 초안이 필요한 자료 | **A (Gamma 등)** | 프롬프트 한 번에 전체 덱. 디자인 고민 0 |
| 회사 표준 템플릿 필수 발표 | **B (PPT/Slides + AI)** | 기존 마스터 슬라이드에 자연스럽게 녹음 |
| 정기적으로 같은 형식의 보고서 PPT | **C (LLM + 자동화)** | 코드로 반복 자동화 가능 |
| 본격적인 외부 IR/세일즈 덱 | **수동 + AI 보조** | 핵심 슬라이드는 사람이 직접 디자인, 나머지만 AI |
| 학습·강의 자료 | **A 또는 B** | 정보 밀도가 높고 시각화 도움 큰 영역 |

→ **핵심**: 자료의 *수명*과 *시청자의 기대치*에 따라 갈래가 결정된다. 일회성·내부용일수록 A, 외부·반복·표준 강제일수록 B나 수동.

## 3. 주요 도구별 비교

### Gamma — 가장 대중적인 AI 네이티브 도구
- 프롬프트 한 줄 → 8~10장짜리 덱 자동 생성
- "한 번에 한 슬라이드"가 아니라 **카드 단위 편집**으로, 길이를 자유롭게 늘릴 수 있음
- 무료 플랜으로도 PDF/PPTX 내보내기 가능
- 한국어 자연스러움 ★★★★☆
- 공식: [gamma.app](https://gamma.app)

### Microsoft Copilot in PowerPoint — 회사용 최강자
- M365 Copilot 라이선스 필요 (유료, 보통 회사가 제공)
- 워드 문서 한 편을 통째로 던지면 **회사 디자인 템플릿에 맞춰 슬라이드 자동 생성**
- "이 슬라이드 다시 디자인해줘", "발표자 노트 만들어줘" 같은 명령형 사용
- 회사 보안·SSO 친화

### Google Slides + Gemini — 워크스페이스 사용자에게 자연스러움
- Google Workspace에서 "Help me create" 또는 한국어로 "이미지 생성"
- 협업·공유가 강점 — 동시 편집 + AI 추천
- 디자인 자유도는 PowerPoint보다 낮지만 학생·스타트업에 충분

### Canva Magic Design — 디자인 감성이 가장 강함
- 비주얼 중심 자료(마케팅·소개 자료·강의 표지)에 압도적
- 한국어 폰트 풀, 일러스트·아이콘 라이브러리가 방대
- PPTX 내보내기 시 일부 애니메이션·동영상 손실 가능

### Tome — 스토리텔링 슬라이드
- 한 장에 한 메시지, 풀스크린 카드 형식
- 영업/제품 데모처럼 **이야기 흐름이 중요한 자료**에 강함
- 일반 보고서용으로는 정보 밀도가 낮음

### Beautiful.ai — 디자인 자동 정렬
- 콘텐츠 입력 시 레이아웃을 **자동으로 다시 조정**
- 컨설팅펌 풍 미니멀 디자인이 기본
- 한국어 폰트 옵션은 제한적

### Plus AI — Google Slides 플러그인
- Slides 안에서 AI로 슬라이드 단위 생성·재디자인
- 기존 Slides 자료를 점진적으로 개선할 때 유용

### Manus, ChatGPT 에이전트 모드 — "에이전트가 직접 PPT 파일을 만든다"
- 2025년 이후 등장한 흐름. AI가 가상 워크스페이스에서 직접 PPTX 파일을 생성해 사용자에게 첨부 파일로 전달
- 아직 디자인 정밀도는 Gamma만 못하지만 **파일 결과물이 PowerPoint 호환**이라는 점이 강력
- 회의록 → PPT처럼 외부 자료 가공이 강점

## 4. 실전 워크플로우 — 빠른 초안에서 발표 마감까지

### 30분 안에 끝내는 초안 (내부 공유용)

1. **메시지 정리** (5분, Claude/ChatGPT): "이 발표에서 청자가 가져갈 핵심 3가지가 뭐지?"라는 질문에 답해본다
2. **Gamma에 프롬프트** (10분):
   ```
   제목: 2026 Q2 OO 프로젝트 진행 현황
   청자: 회사 임원
   톤: 데이터 중심, 8~10장
   포함 내용:
   - 1분기 성과 (매출 +X%)
   - 2분기 목표
   - 리스크 3가지
   - 다음 액션
   ```
3. **카드 단위 수정** (10분): 표현·숫자만 손봄. 디자인은 손대지 말 것 (시간 잡아먹음)
4. **PPTX/PDF 내보내기** (5분)

### 회사 정식 자료 (외부 또는 임원 공유용)

1. **본문 텍스트는 Claude/ChatGPT로 작성** — 톤·논리 흐름 우선
2. **PowerPoint에서 회사 템플릿 열기** → Copilot에 워드/텍스트 붙여넣기
3. Copilot이 **회사 마스터 슬라이드 기반으로 슬라이드 생성**
4. 핵심 1~2 슬라이드만 사람이 직접 디자인 손봄 (표지·결론)
5. 발표자 노트는 Copilot에게 다시 생성 요청

### 반복되는 정기 보고

1. 데이터 소스(스프레드시트·DB)를 정해두고
2. Python + python-pptx로 **템플릿 PPTX에 데이터 주입**하는 스크립트 작성
3. 매주·매월 같은 스크립트로 자동 갱신
4. AI는 *해설 텍스트*만 생성 — 숫자·차트는 코드가 책임

```python
# python-pptx로 매주 같은 형식의 보고서 자동 생성 (개념)
from pptx import Presentation
prs = Presentation("템플릿.pptx")
slide = prs.slides[0]
slide.shapes.title.text = f"주간 보고 — {week}"
# ...
prs.save(f"보고-{week}.pptx")
```

## 5. 프롬프트 팁 — "AI 티"를 줄이려면

좋은 결과를 얻으려면 다음 4가지를 프롬프트에 명시한다.

1. **청자 (Audience)**: "임원", "신입사원", "투자자" 등 — 톤이 완전히 달라진다
2. **목적 (Purpose)**: 의사결정용? 학습용? 소개용?
3. **분량과 구조**: "8장, 표지·목차·결론 포함" 같이 구체적으로
4. **금지사항**: "이모지 사용 금지", "한 슬라이드당 글머리 기호 3개 이내", "스톡 이미지 사용 금지"

나쁜 예: "분기 보고 PPT 만들어줘"
좋은 예:
```
청자: 회사 임원 5명
목적: 2분기 예산 추가 승인 받기
분량: 8장 (표지 + 현황 2장 + 문제 2장 + 제안 2장 + 결론)
톤: 데이터 중심, 감성 표현 배제
금지: 이모지, 일반론적 표현 ("디지털 혁신" 같은 단어)
```

## 6. 함정과 주의점

### "AI가 만든 티"가 그대로 발표 자료로 가는 경우
- 모든 슬라이드가 비슷한 레이아웃, 비슷한 아이콘, 비슷한 색감 → **임원·고객이 즉시 알아챈다**
- 해결: **표지와 결론 슬라이드만큼은 사람이 직접 디자인**

### 환각 (Hallucination)
- AI가 수치·인용·통계를 **그럴듯하게 지어낸다**
- 해결: 숫자가 들어가는 슬라이드는 **반드시 사람이 원 자료와 대조**. AI에게 출처를 함께 적게 한 뒤 검증

### 보안·정보 유출
- 사내 기밀이 들어간 텍스트를 외부 AI 서비스에 그대로 붙여넣으면 **유출 사고**
- 해결: 사내 정책 확인. 가능하면 Microsoft Copilot처럼 **테넌트 내 처리**되는 도구 사용

### 회사 표준 템플릿 무시
- AI 네이티브 도구로 만든 슬라이드는 회사 마스터와 어긋남 → **임원 리뷰에서 반려 위험**
- 해결: 회사 표준이 강제되는 자료는 처음부터 PowerPoint + Copilot 조합

### 텍스트 길이 폭주
- AI가 한 슬라이드에 문장을 너무 많이 채움 → 발표용으로 못 씀
- 해결: 프롬프트에 "한 슬라이드당 글머리 기호 5개 이내, 각 12자 이내" 같은 **하드 룰**을 명시

## 마무리 — 도구가 아니라 "어디까지 사람이 손대는가"가 핵심

AI로 PPT를 만든다는 건 *전부* AI에게 맡긴다는 뜻이 아니다. **초안 생성·디자인 통일·아이콘 배치 같은 잡일은 AI가, 메시지 선택·숫자 검증·표지·결론은 사람이** — 이 분담이 정해지면 도구는 손에 익는 대로 고르면 된다.

내부 자료는 Gamma 한 방에 끝내고, 회사 외부용은 Copilot으로 표준에 맞추고, 정기 보고는 코드로 자동화한다. 그렇게 절약한 시간을 **메시지를 다듬는 데** 다시 쓰는 것이 AI 시대 PPT 작업의 정석이다.

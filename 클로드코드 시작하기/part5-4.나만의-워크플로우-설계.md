여기까지 오시느라 정말 수고하셨습니다. 우리는 클로드코드가 *무엇인지*부터 시작해, *설치하고, 기본기를 다지고, 실전에 써먹고, 나에게 맞게 길들이는* 법까지 모두 배웠어요. 마지막으로, 이 조각들을 **하나의 흐름(워크플로우)** 으로 엮어봅시다.

## 워크플로우 = 내 루틴을 자동화한 흐름

거창하지 않아요. **"내가 반복하는 일의 순서"를 클로드코드에게 맡기는 것** 이 워크플로우입니다. 지금까지 배운 부품들을 떠올려보세요.

- [작업 폴더](part1-3.작업-폴더) — 어디서 일할지
- [좋은 요청](part2-1.좋은-요청-쓰는-법) + [파일 다루기](part2-2.파일-읽고-쓰고-정리시키기) — 무엇을 어떻게
- [CLAUDE.md](part4-1.claude-md-규칙-기억시키기) — 늘 지킬 규칙
- [스킬](part4-2.스킬-단축명령) + [예약](part4-5.예약-자동실행) — 한 번에, 또는 자동으로

이 부품들을 조합하면 나만의 흐름이 됩니다.

![사람이 지휘자처럼 서서, 여러 로봇 비서와 연결된 작업 흐름이 매끄럽게 돌아가는 모습을 흐뭇하게 바라보는 졸업 분위기의 장면](../image/cc-start-part5-4.jpg)

## 예시: '월요일 아침 루틴' 만들기

마케팅 담당자 A씨의 워크플로우를 볼까요.

1. **규칙 정하기** — 업무 폴더에 CLAUDE.md로 "보고서는 개조식, 결과는 report 폴더에"
2. **작업 묶기** — `/주간정리` 스킬: 지난주 회의록 + 지표 파일을 읽어 주간보고 초안 작성
3. **자동 실행** — 매주 월요일 오전 9시에 `/주간정리`가 자동 실행되도록 예약
4. **사람의 몫** — A씨는 자동 생성된 초안을 검토·다듬어 발송

손이 가던 일이 **검토와 판단**만 남는 흐름으로 바뀐 거예요.

## 나만의 워크플로우, 이렇게 시작하세요

1. **반복을 찾기** — "매주/매일 하는 귀찮은 일"을 하나만 골라보세요.
2. **한 번 수동으로** 시켜보며 요청을 다듬기
3. 잘 되면 **CLAUDE.md·스킬로 굳히기**
4. 안정되면 **예약으로 자동화**
5. 익숙해지면 **다음 반복 작업**으로 확장

> 💡 처음부터 완벽한 시스템을 만들려 하지 마세요. **하나의 작은 루틴**부터 — 그게 진짜 시작입니다.

## 그리고, 여기서부터는 여러분의 차례

이 과정은 '시작하기'였어요. 진짜 실력은 **직접 시켜보며** 늘어납니다. 오늘 당장, 미뤄뒀던 작은 일 하나를 클로드코드에게 맡겨보세요. 막히면 [물어보면](part5-3.잘-안될-때-대처법) 되고, 실수해도 [되돌리면](part2-4.되돌리기) 됩니다.

여러분은 이미 가장 큰 고비 — **'코딩 도구'라는 두려움** — 을 넘으셨으니까요. 🙌

## 오늘의 핵심 한 줄

> **반복 → 다듬기 → 규칙·스킬로 굳히기 → 자동화. 작은 루틴 하나부터 나만의 워크플로우가 된다.**

여기까지 함께해주셔서 감사합니다. 이제, 여러분만의 비서를 마음껏 부려보세요. 🎉

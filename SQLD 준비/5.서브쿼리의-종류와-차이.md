## 1. 왜 헷갈리는가

"서브쿼리"라는 말 자체는 하나지만, 실제로는 **반환하는 결과의 모양**과 **바깥 쿼리와의 관계**에 따라 완전히 다른 문법 규칙이 적용된다. 이걸 구분 못 하면 "왜 이 서브쿼리는 `=`를 못 쓰고 `IN`을 써야 하는가" 같은 질문에서 막힌다.

## 2. 반환 결과 모양에 따른 분류

| 종류 | 반환 값 | 사용 가능한 연산자 |
|---|---|---|
| 단일행 서브쿼리 | 1행 1컬럼 | `=`, `>`, `<`, `>=`, `<=`, `<>` |
| 다중행 서브쿼리 | 여러 행 1컬럼 | `IN`, `ANY`, `ALL`, `EXISTS` |
| 다중컬럼 서브쿼리 | 1행 이상, 여러 컬럼 | `IN` (컬럼 묶음 비교) |

```sql
-- 단일행: 특정 부서 평균급여보다 많이 받는 직원
SELECT 이름 FROM 직원
WHERE 급여 > (SELECT AVG(급여) FROM 직원 WHERE 부서 = '개발팀');

-- 다중행: 서울에 있는 모든 부서 소속 직원
SELECT 이름 FROM 직원
WHERE 부서번호 IN (SELECT 부서번호 FROM 부서 WHERE 지역 = '서울');
```

가장 흔한 오답 패턴: 다중행이 반환될 수 있는 서브쿼리에 `=`을 쓰는 것이다. 실행 시점에 결과가 2행 이상이면 "단일 행 서브쿼리에 2개 이상의 행이 반환되었습니다" 에러가 난다. 서브쿼리가 조건에 따라 몇 행이 나올지 모른다면 `=` 대신 `IN`을 쓰는 게 안전하다.

## 3. ANY와 ALL의 차이

다중행 서브쿼리에서 비교 연산자를 쓰고 싶을 때 ANY/ALL을 쓴다. 방향을 헷갈리기 쉽다.

```sql
-- ANY: 서브쿼리 결과 중 하나라도 만족하면 참 (즉, 최솟값보다 크면 참)
WHERE 급여 > ANY (SELECT 급여 FROM 직원 WHERE 부서 = '영업팀')

-- ALL: 서브쿼리 결과 전부를 만족해야 참 (즉, 최댓값보다 커야 참)
WHERE 급여 > ALL (SELECT 급여 FROM 직원 WHERE 부서 = '영업팀')
```

`> ANY`는 "가장 작은 값보다만 크면 됨"이고, `> ALL`은 "가장 큰 값보다도 커야 함"이다. 부등호 방향이 바뀌면 최댓값/최솟값 기준도 반대로 뒤집힌다는 걸 표로 외워두면 헷갈리지 않는다.

## 4. 연관 서브쿼리(Correlated Subquery) vs 비연관 서브쿼리

이게 SQLD에서 가장 많이 틀리는 포인트다.

- 비연관 서브쿼리: 서브쿼리가 바깥 쿼리와 무관하게 **독립적으로 먼저 한 번 실행**되고, 그 결과를 바깥 쿼리가 사용한다.
- 연관 서브쿼리: 서브쿼리 안에서 바깥 쿼리의 컬럼을 참조한다. 그래서 **바깥 쿼리의 행마다 서브쿼리가 반복 실행**된다.

```sql
-- 비연관: 서브쿼리 먼저 1번 실행 → 전체 평균급여 하나 확정 → 바깥 쿼리가 비교
SELECT 이름 FROM 직원
WHERE 급여 > (SELECT AVG(급여) FROM 직원);

-- 연관: 바깥 쿼리(직원 e)의 각 행마다, 그 직원의 부서로 서브쿼리가 다시 실행됨
SELECT 이름 FROM 직원 e
WHERE 급여 > (SELECT AVG(급여) FROM 직원 WHERE 부서 = e.부서);
```

연관 서브쿼리는 "같은 부서 내에서만" 비교하는 것처럼, 바깥 쿼리 행마다 조건이 달라지는 로직을 짤 때 쓴다. 대표적으로 `EXISTS`는 거의 항상 연관 서브쿼리와 함께 쓰인다.

```sql
-- 주문 이력이 있는 고객만 조회 (EXISTS는 연관 서브쿼리로 짜는 게 일반적)
SELECT 고객명 FROM 고객 c
WHERE EXISTS (SELECT 1 FROM 주문 o WHERE o.고객번호 = c.고객번호);
```

## 5. 시험 함정 정리

- "서브쿼리는 무조건 WHERE절에만 쓴다" → SELECT절(스칼라 서브쿼리), FROM절(인라인 뷰), HAVING절 어디든 올 수 있다.
- "EXISTS는 서브쿼리 결과값을 직접 비교한다" → 아니다. EXISTS는 서브쿼리가 **행을 반환하는지 여부(존재 유무)**만 본다. 실제 값이 아니라 존재 자체가 조건이라, `SELECT 1`처럼 아무 값이나 넣어도 결과는 동일하다.

## 1. 정의

둘 다 두 개 이상의 SELECT 결과를 세로로 합치는 집합 연산자다. 차이는 하나뿐이다.

- UNION: 결과를 합친 뒤 **중복 행을 제거**한다.
- UNION ALL: 중복을 제거하지 않고 **있는 그대로 다 합친다**.

```sql
SELECT 이름 FROM 정직원
UNION
SELECT 이름 FROM 계약직원;
-- 양쪽에 이름이 겹치면 한 번만 나온다

SELECT 이름 FROM 정직원
UNION ALL
SELECT 이름 FROM 계약직원;
-- 겹쳐도 각각 다 나온다 (총 행수 = 양쪽 행수의 합)
```

## 2. 왜 UNION ALL이 더 빠른가

UNION은 중복 제거를 위해 내부적으로 **정렬(SORT) 또는 해시 연산**을 거쳐야 한다. 데이터가 많을수록 이 비용이 커진다. UNION ALL은 이 과정이 없으므로 훨씬 빠르다.

**중복이 발생할 수 없다는 것을 아는 상황(예: 서로 다른 연도의 데이터를 합칠 때)이라면 UNION ALL을 써야 한다.** 습관적으로 UNION을 쓰는 것은 SQLD에서도 실무에서도 감점 포인트다.

## 3. 사용 조건

두 SELECT문은 다음을 만족해야 한다.

- 컬럼 개수가 같아야 한다.
- 대응하는 컬럼끼리 데이터 타입이 호환되어야 한다.
- 컬럼명은 **첫 번째 SELECT문 기준**으로 결정된다.
- ORDER BY는 전체 결과 맨 마지막에 한 번만 쓸 수 있다.

```sql
SELECT 이름, 급여 FROM 정직원
UNION ALL
SELECT 이름, 급여 FROM 계약직원
ORDER BY 급여 DESC;   -- 마지막 SELECT 뒤에 한 번만
```

## 4. INTERSECT / MINUS(EXCEPT)와의 관계

집합 연산자는 UNION 계열만 있는 게 아니다. 함께 묶어서 이해해야 헷갈리지 않는다.

| 연산자 | 의미 | 비유 |
|---|---|---|
| UNION | 합집합 (중복 제거) | A ∪ B |
| UNION ALL | 합집합 (중복 유지) | A + B 그대로 |
| INTERSECT | 교집합 | A ∩ B |
| MINUS (Oracle) / EXCEPT (표준, SQL Server) | 차집합 | A - B |

```sql
-- 양쪽 부서에 모두 소속된 사람 (교집합)
SELECT 이름 FROM 영업부소속
INTERSECT
SELECT 이름 FROM 프로젝트팀소속;

-- 영업부에만 있고 프로젝트팀엔 없는 사람 (차집합)
SELECT 이름 FROM 영업부소속
MINUS
SELECT 이름 FROM 프로젝트팀소속;
```

INTERSECT와 MINUS는 UNION ALL처럼 "있는 그대로"라는 옵션이 없다 — 둘 다 결과에서 중복을 제거한 상태로 반환한다는 점도 시험에 자주 나온다.

## 5. 시험 함정 정리

- "UNION ALL은 정렬 안 하니까 순서가 항상 랜덤이다" → 정렬을 안 한다는 것과 순서가 랜덤이라는 것은 다른 이야기다. 명시적 ORDER BY가 없으면 순서를 보장하지 않을 뿐, 보통은 SELECT 순서대로(위 결과 먼저, 아래 결과 다음) 나온다.
- "MINUS는 A-B와 B-A가 같다" → 차집합은 순서에 따라 결과가 다르다. 교환법칙이 성립하지 않는다.

## 1. 정의

CROSS JOIN은 두 테이블의 **모든 행을 조건 없이 곱한 결과(카티션 곱, Cartesian Product)**를 반환한다. A 테이블이 3행, B 테이블이 4행이면 결과는 항상 12행이다. 조인 조건(ON)이 없다는 것이 핵심이다.

```sql
SELECT *
FROM 부서 CROSS JOIN 직원;
-- 부서 5행, 직원 20행이면 결과는 100행
```

## 2. INNER JOIN과 뭐가 다른가

INNER JOIN도 내부적으로는 "카티션 곱을 만든 뒤 조건에 맞는 행만 남긴다"는 논리적 순서를 따른다. 그래서 초보자가 "ON 조건 없는 INNER JOIN = CROSS JOIN 아닌가?"라고 헷갈린다. 실제로 맞다.

```sql
-- 이 둘은 결과가 같다
SELECT * FROM A CROSS JOIN B;
SELECT * FROM A, B;              -- FROM절에 콤마로 나열 + WHERE 조건 없음
SELECT * FROM A INNER JOIN B ON 1=1;
```

차이는 **의도의 표현**이다. INNER JOIN은 ON 조건으로 "관계"를 명시하는 문법이고, CROSS JOIN은 애초에 조건이 없다는 것을 명시적으로 선언하는 문법이다. SQLD 문제에서는 "ON 절이 없는 INNER JOIN"과 "CROSS JOIN"이 같은 결과를 낸다는 것을 이용해 오답 선지를 만든다.

## 3. 행 개수로 구분하는 법

시험에서 자주 나오는 함정: "다음 SQL의 결과 행 수는?"

| 조인 종류 | 결과 행 수 | 조건 |
|---|---|---|
| CROSS JOIN | A행수 × B행수 | 항상 |
| INNER JOIN (조건 있음) | A행수 × B행수 이하 | 조건 만족하는 조합만 |
| LEFT OUTER JOIN | A행수 이상 | A의 모든 행 보존, 매칭 없으면 NULL |
| FULL OUTER JOIN | A∪B 기준 | 양쪽 미매칭 행 모두 보존 |

CROSS JOIN만 유일하게 결과 행 수가 **입력 행 수만으로 정확히 계산 가능**하다. 다른 조인은 조건에 따라 달라지므로, 문제에서 "정확한 행 수를 구할 수 있는 것은?"이라고 물으면 CROSS JOIN이 답인 경우가 많다.

## 4. 실무에서 언제 쓰는가

- 모든 조합을 미리 만들어야 하는 경우 (예: 날짜 테이블 × 상품 테이블로 "상품별 일자별" 기본 틀 생성)
- 테스트 데이터 생성

## 5. 시험 함정 정리

- "CROSS JOIN에는 ON 절을 쓸 수 없다" → 표준 CROSS JOIN 문법은 ON을 쓰지 않는다. WHERE로 조건을 추가하면 결과적으로 INNER JOIN과 동일해진다.
- "CROSS JOIN 결과는 항상 INNER JOIN보다 많다" → 조인 조건이 전혀 없는 INNER JOIN(`ON 1=1`)과는 같을 수 있다. "많거나 같다"가 정확한 표현.

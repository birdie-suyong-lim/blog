## 1. NULL은 "값이 아니다"

NULL은 0도, 빈 문자열도 아니다. **"알 수 없음(unknown)" 또는 "정의되지 않음"**을 뜻하는 상태다. 그래서 NULL이 관여하는 모든 비교/연산은 참/거짓이 아니라 **UNKNOWN**이라는 제3의 결과를 낸다. SQL은 TRUE/FALSE만 있는 2치 논리가 아니라 TRUE/FALSE/UNKNOWN의 **3치 논리(Three-Valued Logic)**로 동작한다.

```sql
SELECT * FROM 직원 WHERE 보너스 = NULL;   -- 결과: 항상 0건
```

`보너스 = NULL`은 문법 오류가 아니다. 하지만 결과는 절대 참이 될 수 없다. NULL은 "알 수 없는 값"이므로 "알 수 없는 값이 NULL과 같은가?"라는 질문 자체가 답을 낼 수 없는(UNKNOWN) 질문이기 때문이다. 그래서 반드시 `IS NULL` / `IS NOT NULL`을 써야 한다.

```sql
SELECT * FROM 직원 WHERE 보너스 IS NULL;      -- 올바른 방법
SELECT * FROM 직원 WHERE 보너스 IS NOT NULL;
```

## 2. WHERE절은 결국 TRUE인 행만 남긴다

WHERE의 조건 평가 결과가 FALSE든 UNKNOWN이든, **TRUE가 아니면 그 행은 걸러진다.** 이 규칙 하나로 NULL 관련 함정 문제 대부분을 풀 수 있다.

```sql
SELECT * FROM 직원 WHERE 보너스 > 100 OR 보너스 <= 100;
```

보너스가 NULL인 행이 있다면, 직관적으로는 "저 두 조건 중 하나는 항상 참 아닌가?"(배중률) 싶지만, 보너스가 NULL이면 두 비교 모두 UNKNOWN이 되고 `UNKNOWN OR UNKNOWN`도 UNKNOWN이라 그 행은 결과에서 빠진다.

## 3. 3치 논리 진리표

| A | B | A AND B | A OR B |
|---|---|---|---|
| TRUE | UNKNOWN | UNKNOWN | TRUE |
| FALSE | UNKNOWN | FALSE | UNKNOWN |
| UNKNOWN | UNKNOWN | UNKNOWN | UNKNOWN |

AND는 "하나라도 FALSE면 무조건 FALSE"(UNKNOWN을 이긴다), OR는 "하나라도 TRUE면 무조건 TRUE"(UNKNOWN을 이긴다)는 규칙만 기억하면 표를 다 외울 필요는 없다.

## 4. NULL 관련 함수로 안전하게 다루기

| 함수 | 동작 |
|---|---|
| `NVL(a, b)` (Oracle) / `ISNULL(a, b)` (SQL Server) | a가 NULL이면 b로 치환 |
| `COALESCE(a, b, c, ...)` | 표준 함수. 앞에서부터 NULL이 아닌 첫 값 반환 |
| `NULLIF(a, b)` | a와 b가 같으면 NULL, 다르면 a 반환 |

```sql
-- 보너스가 NULL이면 0으로 취급해서 급여+보너스 계산
SELECT 이름, 급여 + COALESCE(보너스, 0) AS 총액 FROM 직원;
```

`급여 + 보너스`처럼 산술 연산에 NULL이 하나라도 섞이면 결과 전체가 NULL이 된다는 것도 자주 나오는 함정이다. `COALESCE`/`NVL`로 먼저 감싸지 않으면 보너스가 없는 직원은 총액 자체가 NULL로 나온다.

## 5. 집계함수와 NULL

`COUNT(*)`는 NULL을 포함한 모든 행을 센다. 반면 `COUNT(컬럼명)`은 해당 컬럼이 NULL인 행을 제외하고 센다. `SUM`, `AVG`, `MAX`, `MIN`도 전부 NULL을 계산에서 제외한다(0으로 취급하지 않는다).

```sql
SELECT COUNT(*), COUNT(보너스), AVG(보너스) FROM 직원;
-- COUNT(*): 전체 인원
-- COUNT(보너스): 보너스가 NULL이 아닌 인원
-- AVG(보너스): 보너스가 있는 사람들만의 평균 (전체 인원으로 나누지 않음)
```

## 6. 시험 함정 정리

- "`<> NULL`이면 NULL이 아닌 행이 나온다" → 틀림. `<>` 비교도 NULL이 개입하면 결과는 UNKNOWN이라 아무 행도 나오지 않는다.
- "빈 문자열(`''`)과 NULL은 같다" → Oracle은 빈 문자열을 NULL로 취급하지만, 이는 Oracle의 특이 동작이고 SQL 표준이나 SQL Server 등에서는 별개의 값이다. SQLD는 표준/Oracle 문제가 섞여 나오므로 문제의 DBMS 전제를 확인해야 한다.
- "AVG(보너스)는 보너스 없는 직원까지 포함해 계산한다" → 아니다. NULL은 평균 계산의 분모(건수)에서 제외된다.
